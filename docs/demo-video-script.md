# 医生演示视频脚本

建议录制 3 到 5 分钟 MP4，分辨率 1920x1080。录制时使用本地 MTDDH 公开/脱敏样例或其他确认可展示的示例图片，不使用医院真实隐私数据，也不把甲方 X 光片上传到 GitHub。

## 画面 1：准备数据集

旁白：
“标注前，请先把本次要标注的 X 光图片统一放到一个文件夹中。文件夹里可以放 jpg、png、bmp 等常见图片格式。如果图片命名或目录很混乱，可以先发给我们整理。”

操作：
打开一个示例文件夹，展示多张待标注图片都在同一目录下。

## 画面 2：启动工具

旁白：
“解压工具后，双击 Hip22AnnotationTool.exe。程序会在本机运行，并自动打开浏览器标注界面。”

操作：
双击 exe，展示浏览器打开 `127.0.0.1` 页面。

## 画面 3：导入文件夹

旁白：
“点击导入文件夹，选择刚才准备好的图片文件夹。导入后，系统会用增强图自动识别文件夹中的图片，并在后台生成初始关键点。”

操作：
点击“导入文件夹” -> “选择” -> 选中示例文件夹 -> “导入”。展示底部缩略图和自动识别进度。
同时展示右侧“文件夹进度”和状态筛选，说明未标注、待复核、修改中、已完成的含义。

## 画面 4：拖拽修改标注点

旁白：
“自动识别结果只是初始参考。医生需要逐张检查，如果点位不准确，直接拖拽到正确位置。拖拽后的点会记录为人工修改。”

操作：
打开一张图，放大局部，拖拽 1 到 2 个点位。可以展示左侧点位列表随状态变化。
如果示例图片没有自动点，可点击顶部“自动识别”重试，并说明该按钮会保留人工修改点；仍识别不出来时继续手工放点。

### 可选补充镜头：增强预览、ROI、扫描四角与 Shenton 研究标注

旁白：
“打开图片后默认显示原图，增强图用于辅助观察泪滴或骨性边界。遇到手机拍摄背景或杂乱边框时，可以框选 ROI 后在关注区域内重试识别。如果能看到片子四个边角，也可以使用扫描四角，让系统先做类扫描透视校正再识别。Shenton 曲线点画完后仍可拖动调整；右侧基础测量目前用于研究复核，不是临床诊断结论。”

操作：
切换“增强图 / 原图”；点击 “ROI” 画一个关注区域并使用“适配 ROI”放大查看；点击“扫描”按左上、右上、右下、左下点四个片子角，说明识别点仍会映射回原图坐标；再点击 “Shenton”，在闭孔上缘和股骨颈内下缘各点至少 3 个点，并说明医生可以沿轮廓标更多点让曲线更贴合，展示右侧 Shenton、AI/Tonnis、Sharp、CE、颈干角和髋臼深度状态。

## 画面 5：保存

旁白：
“修改完成后点击保存。工具默认也会自动保存，并在切换图片前保存当前图片。”

操作：
点击“保存”，展示状态变为“已保存”。打开文件夹，展示 `annotations` 子目录和同名 `.txt` 文件已生成。
展示右侧最近保存时间，以及文件夹内自动生成的 `HIP22_STATUS_DONE_..._TODO_....txt` 和 `HIP22_status_report.html`。

## 画面 6：提交

旁白：
“全部图片完成后，请把整个标注完成的文件夹发给我们即可。不要只发单个结果文件。完整流程是：准备数据集，导入文件夹，自动识别，拖拽修改标注点，保存，将整个文件夹发送给我们。”

操作：
回到文件夹视图，框选整个文件夹或展示压缩发送动作。
强调如果界面提示目录嵌套、命名混乱或重复文件名，可先把照片发给项目团队整理。
